# 🐛 调试功能使用指南

## 快捷键

### 调试模式控制
- **F1**: 开启/关闭调试模式
- **F2**: 显示/隐藏块边界(灰色网格)
- **F3**: 显示/隐藏脏块(红色高亮)

## 调试可视化说明

### 1. 块边界显示 (F2)
- **灰色网格**: 显示64×64像素的块边界
- **用途**: 查看块的划分情况,定位边界问题

### 2. 脏块显示 (F3)
- **红色高亮**: 显示当前帧被更新的块
- **用途**:
  - 查看哪些块被标记为脏
  - 检查脏矩形优化是否正常工作
  - 诊断块之间的同步问题

## 使用场景

### 场景1: 检查边界线条问题
1. 按**F1**开启调试模式
2. 按**F2**显示块边界
3. 按**F3**显示脏块
4. 绘制沙子或水,观察它们跨块移动时:
   - 块边界是否正确显示
   - 脏块是否正确标记
   - 边界处是否有视觉瑕疵

### 场景2: 检查脏矩形优化
1. 按**F3**显示脏块
2. 在小范围内绘制材料
3. 观察:
   - 只有包含材料的块被标记为脏(红色)
   - 空块不会被更新(优化生效)
   - 材料移动时,脏块标记跟随移动

### 场景3: 检查跨块移动
1. 按**F2**和**F3**同时开启
2. 在块边界附近绘制沙子
3. 观察沙子下落时:
   - 沙子从一个块移动到另一个块
   - 两个块都被标记为脏(红色)
   - 边界处没有像素丢失或重复

## 预期行为

### 正常情况
- **脏块标记**: 只有包含活跃像素的块被标记为脏
- **跨块移动**: 当像素跨块移动时,源块和目标块都被标记为脏
- **边界同步**: 边界处没有视觉瑕疵,像素连续流动

### 异常情况诊断

#### 问题1: 边界处有线条
- **现象**: 块边界处有明显的线条或断裂
- **可能原因**:
  - 边界同步不完整
  - 脏块标记不正确
- **检查方法**:
  - 开启F3,观察边界两侧的块是否都被标记为脏
  - 如果只有一侧被标记,说明跨块移动有问题

#### 问题2: 像素消失
- **现象**: 像素在边界处消失
- **可能原因**:
  - 跨块移动失败
  - 边界同步覆盖了像素
- **检查方法**:
  - 开启F3,观察像素移动路径上的块是否都被标记为脏
  - 如果某个块没有被标记,说明跨块移动没有触发

#### 问题3: 性能问题
- **现象**: 所有块都被标记为脏,即使只有少量像素
- **可能原因**:
  - 脏矩形扩展过度
  - 边界同步触发了所有块
- **检查方法**:
  - 开启F3,观察红色块的数量
  - 如果整个世界都是红色,说明优化失效

## 性能影响

- **调试模式**: 对性能影响很小(仅增加渲染开销)
- **块边界显示**: 轻微影响(绘制网格线)
- **脏块显示**: 中等影响(每帧检查所有块状态)

建议在调试时开启,正常使用时关闭。

```

## 示例测试流程

1. 启动demo
2. 按**F1**开启调试模式
3. 按**F2**和**F3**显示块边界和脏块
4. 按**1**运行密度测试
5. 观察:
   - 沙子、油、水是否正确分层
   - 材料跨块移动时,块是否正确标记为脏
   - 边界处是否有视觉问题
6. 使用鼠标在块边界附近绘制材料
7. 观察跨块移动的效果

    return dirty_rect_.is_dirty;
}
```

if (chunk && chunk->is_dirty()) {
    SDL_SetRenderDrawColor(renderer, 255, 0, 0, 80);  // 红色半透明
