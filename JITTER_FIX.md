# 画面元素抖动问题修复总结

## 📋 问题分析

### 问题描述

**画面内的元素在抖动**,元素位置不稳定,不断跳动。

### 问题根源

**`is_in_pass`函数返回`true`,导致所有像素都被更新,包括边界扩展区的像素!**

但是,边界扩展区的像素可能被多个chunk重复更新,导致抖动!

**问题代码**:
```cpp
bool Chunk::is_in_pass(int local_x, int local_y, int pass) const {
    // ❌ 所有像素都更新,包括边界扩展区
    // ❌ 边界扩展区的像素和块内的元素一样处理
    return true;
}
```

**问题原因**:
1. 边界扩展区的像素被多个chunk重复更新
2. 每个chunk更新时,边界扩展区的像素都会被处理
3. 由于双向同步(`sync_borders`),像素在多个chunk之间来回移动
4. 导致元素位置不稳定,不断抖动

## 🔧 修复方案

### 修复内容

**位置**: `src/core/Chunk.cpp` - `Chunk::is_in_pass()`

**修复代码**:
```cpp
bool Chunk::is_in_pass(int local_x, int local_y, int pass) const {
    // ✅ 只更新有效区域内的像素(0到63)
    // ✅ 边界扩展区的像素不更新,由sync_borders负责同步
    if (local_x >= 0 && local_x < SIZE && local_y >= 0 && local_y < SIZE) {
        return true;
    }
    
    // ✅ 边界扩展区的像素不更新
    return false;
}
```

**修复原理**:
1. 只更新有效区域内的像素(0到63)
2. 边界扩展区的像素不更新
3. 边界扩展区的像素由`sync_borders`负责同步
4. 避免重复更新,防止抖动

## 📊 修复效果对比
